Java's development and execution involve three key components: JVM, JRE, and JDK. Here's what each one does:

# 1. JVM (Java Virtual Machine)
What it does:
- The JVM is a runtime environment that executes Java bytecode.
- It interprets the compiled .class files and converts them into machine code specific to the operating system.

## Key Features:

- Platform Independence: JVM makes Java platform-independent.
- Garbage Collection: Automatically manages memory.
- Security: Provides a secure environment for code execution.

# 2. JRE (Java Runtime Environment)
What it does:
- The JRE provides all the necessary libraries and JVM to run Java applications.

## Key Features:

- Contains the JVM and essential libraries.
- Can only run Java applications; cannot compile them.
  
Who uses it:
- End-users who want to run Java programs.

# 3. JDK (Java Development Kit)
What it does:
- The JDK is a complete development kit that includes tools for developing, debugging, and running Java applications.

## Key Features:

- Includes the JRE, compiler (javac), and tools like javadoc, jdb, etc.
- Used to write and compile Java programs.
  
Who uses it:
- Developers who create Java applications.

# Relationship 
- JDK = JRE + Development Tools
- JRE = JVM + Libraries
- JVM = Core Component to Execute Bytecode

# QUICK ANALOGY
- JVM: A virtual engine that drives the car.
- JRE: The car with everything needed to drive it.
- JDK: A workshop where you can build and maintain the car.
